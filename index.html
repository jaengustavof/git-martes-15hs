Areas de GIT

Working Area

Area de trabajo hace referencia a la carpeta donde creamos el repositorio. Muy rar vez utilizaremos el area de comandos en esta zona, pero es donde ralizaremostodas las modificaciones relacionadas con nuestro proyecto

Staging Area (INDEX)

El area de control de cambios corresponde al archivo que se encuentra en la carpeta .git que se crea al iniciar un repositorio. Aqui se registran los archivos a los que vamos a hacerle seguimiento y es el punto de comparacion para confirmar dichis cambios. Es necesario agregar los archivos con los que estemos trabajando a dicha area para avanzar con el historial de cambios, ya que git al utilizar el comandos git addrealiza una captura de pantalla.

Repository (Local/Remote)

El repositorio corresponde a la base de datod donde se almacenan los cambios en formato BLOB(Binary Object File) y es el lugar al que podemos acceder cada vez que necesitemos revisar un estado anterior de nuestro proyecto. Esto es posible gracias de los commits identificados mediante un numero de HASH.
